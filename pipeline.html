PROVER · runs locally, holds secrets

Has private inputs (the witness). Runs the .wasm to compute every signal, then uses the .zkey to produce a proof. Heavy compute, runs once per claim.

VERIFIER · runs anywhere, public

Has only the verification key + the public inputs + the proof. Outputs true or false. Cheap, succinct — milliseconds, kilobytes.

The seven commands that ship a proof

Bookmark this — it’s the entire workflow.

# 1. compile circuit → r1cs + wasm
circom circuit.circom --r1cs --wasm --sym

# 2. setup phase 2 (uses ptau from pre-flight)
snarkjs groth16 setup circuit.r1cs powersOfTau28_hez_final_14.ptau circuit_0000.zkey

# 3. contribute entropy (each prover adds randomness)
snarkjs zkey contribute circuit_0000.zkey circuit_final.zkey --name="me" -v

# 4. export the verification key (public, tiny, reused forever)
snarkjs zkey export verificationkey circuit_final.zkey verification_key.json

# 5. compute witness from your private inputs
node circuit_js/generate_witness.js circuit_js/circuit.wasm input.json witness.wtns

# 6. produce the proof
snarkjs groth16 prove circuit_final.zkey witness.wtns proof.json public.json

# 7. verify (anyone can do this — no secrets needed)
snarkjs groth16 verify verification_key.json public.json proof.json
