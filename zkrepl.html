Meet zkrepl.dev — compile and prove in your browser, no toolchain required.

zkrepl.dev — multiplier.circom 
pragma circom 2.1.6;

template Multiplier() {
    signal input  a;
    signal input  b;
    signal output c;
    c <== a * b;
}

component main {public [a]} = Multiplier();

/* INPUT = { "a": "3", "b": "11" } */

Why we use it for teaching

  • Compiles & proves in your browser — no toolchain.
  • Inputs live in a comment — share circuits as a single file.
  • Built-in circomlib — include "circomlib/poseidon.circom"; just works.
  • Include any file from GitHub via raw URL — see example below.
  • Constraint count printed live — feel the cost of your code.
  • ⌘ Enter to compile & prove.
CAVEAT

zkREPL silently does input % p on every input, so it cannot reproduce overflow bugs. Real snarkjs will. Don’t skip the local toolchain forever.

Using circomlib

zkREPL ships with circomlib ↗ pre-loaded — no install, no path setup. Just include and use. Here: Poseidon hash of two field elements, the hash function used throughout the ZK ecosystem.

zkrepl.dev — poseidon_hash.circom ⌘ Enter to run 
pragma circom 2.1.6;

include "circomlib/poseidon.circom";

// Poseidon(n) hashes n field elements into one.
// It is ZK-friendly: far cheaper in constraints than SHA-256.

template HashTwo() {
    signal input  left;
    signal input  right;
    signal output hash;

    component h = Poseidon(2);
    h.inputs[0] <== left;
    h.inputs[1] <== right;
    hash <== h.out;
}

component main {public [left, right]} = HashTwo();

/* INPUT = { "left": "1", "right": "2" } */

✓ compiled · 240 constraints
✓ witness ok · hash = 7853200120776062878684798364095072458815029376092732009249414926327459813530
✓ groth16 proof generated

Including libraries from GitHub

zkREPL can fetch any .circom file via a raw GitHub URL. Here: 256-bit modular multiplication using 0xPARC/circom-secp256k1 ↗, with the secp256k1 prime encoded as 4 × 64-bit little-endian limbs.

zkrepl.dev — bigmult.circom ⌘ Enter to run 
pragma circom 2.1.6;

include "https://raw.githubusercontent.com/0xPARC/circom-secp256k1/master/circuits/bigint.circom";

// BigMultModP(n, k): computes (a * b) mod p
// where a, b, p are k-limb numbers with each limb n bits wide.
// For (64, 4): 256-bit numbers as 4 × 64-bit limbs (little-endian).

template Example() {
    signal input  a[4];  // 256-bit input split into 4 × 64-bit limbs
    signal input  b[4];
    signal output out[4];

    // secp256k1 prime p = 2^256 - 2^32 - 977
    // in 4 × 64-bit little-endian limbs:
    var p[4] = [
        18446744069414583631,  // p[0] = 2^64 - 2^32 - 977
        18446744073709551615,  // p[1] = 2^64 - 1
        18446744073709551615,  // p[2] = 2^64 - 1
        18446744073709551615   // p[3] = 2^64 - 1
    ];

    component mult = BigMultModP(64, 4);

    for (var i = 0; i < 4; i++) {
        mult.a[i] <== a[i];
        mult.b[i] <== b[i];
        mult.p[i] <== p[i];
    }

    for (var i = 0; i < 4; i++) {
        out[i] <== mult.out[i];
        log("out[", i, "] =", mult.out[i]);
    }
}

component main {public [a, b]} = Example();

/* INPUT = {
    "a": ["3", "0", "0", "0"],
    "b": ["7", "0", "0", "0"]
} */
